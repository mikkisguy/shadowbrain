Sidebar Navigation

Classic app layout. Persistent left sidebar with navigation. Main content area shows the selected view. Search bar always visible at top of content.

Pros

  • Familiar, works well on desktop
  • Easy to add more sections later
  • Clear sense of place

Cons

  • Sidebar takes space on smaller screens
  • More nested navigation to manage

Single-Page Feed

Minimalist, feed-forward. One main view with inline filters. Everything happens on the same page. Type tabs for quick filtering.

Pros

  • Simple, focused
  • Works great on mobile
  • Fast to build
  • Matches " Hermes is primary" philosophy

Cons

  • Harder to add dedicated views later
  • Less sense of "place"
  • Settings/tags need separate pages

Top Navigation

Section-based navigation. Top nav bar with dedicated pages for Browse, Search, Graph, Tags, Settings. Each page is its own view.

Pros

  • Room for dedicated features per page
  • Full width for content
  • Clear URL structure (/graph, /tags)
  • Scales well as features grow

Cons

  • More pages to build
  • More navigation clicks
  • Top nav can get crowded
