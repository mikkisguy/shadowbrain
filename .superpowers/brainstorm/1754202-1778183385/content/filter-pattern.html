How should filters work with infinite scroll?

Different approaches have different trade-offs

ShadowBrain
All Notes Journal + Tags

Sticky Filters (Top of Feed)

Filters stay visible at the top as you scroll. Changing a filter resets the feed and starts fresh from the top.

Pros

  • Filters always accessible
  • Clear what's active
  • Easy to switch contexts

Cons

  • Loses scroll position on filter change
  • Sticky header takes up space
  • "Resets feed" can feel jarring
ShadowBrain

No Inline Filters (Search Instead)

Browse shows everything chronologically. Use the Search page for filtering by type, tags, dates. Browse = pure timeline.

Pros

  • Simple, focused
  • Pure chronological flow
  • Search page can be more powerful
  • No scroll position issues

Cons

  • Can't quickly filter to just notes
  • Need to leave page to filter
  • Browse might feel cluttered

Type Tabs Only (Tags in Search)

Quick type tabs at the top (All/Notes/Journal/Bookmarks). Tags and advanced filters live on the Search page. Balance of convenience and simplicity.

Pros

  • Type filtering is the common case
  • Easy to switch between content types
  • Search page handles complex queries
  • Keeps Browse clean

Cons

  • Can't filter by tag quickly
  • Still resets scroll on tab change
  • Two places for filtering logic
